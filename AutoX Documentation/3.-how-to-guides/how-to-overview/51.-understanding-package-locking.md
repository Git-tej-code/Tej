# Understanding Package Locking

**Who can do this**

* Platform Admin
* Power User (with deployment permission)

&#x20;

**When to Lock a Package**

* Package is ready for promotion
* No further changes are required
* Testing is complete

&#x20;

**Steps**

1. Open the Package
2. Review:
3. Process list
4. Versions included
5. Click Lock
6. Confirm action

&#x20;

**What Happens Next**

* Package becomes immutable
* Package is eligible for deployment
* Editing is disabled

&#x20;

**Important Notes**

* Locking is mandatory before deployment
* Locked packages cannot be edited
* Locking does not deploy the package
