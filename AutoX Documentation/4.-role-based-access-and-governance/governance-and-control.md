# Governance & Control

**Versioning Governance**

**What Is Governed**

* Processes
* Activities
* Agents
* Packages

&#x20;

**Governance Rules**

* Every change creates a new version
* Locked versions are immutable
* No deletion of core artefacts
* Rollback is achieved via redeployment

***

**Deployment Governance**

Rules

* Deployment is package-level only
* Packages must be locked
* Environment transitions are enforced
* All deployments are audited

***

**Audit Logs**

**What Is Captured**

* User actions
* Role changes
* Configuration updates
* Deployment events

&#x20;

**Why This Matters**

* Compliance
* Accountability
* Incident investigation

***

**System Performance & Observability Governance**

**What Is Monitored**

* Execution latency
* Cost consumption
* Agent performance
* Failure rates

&#x20;

**Governance Outcome**

* Early detection of issues
* Cost control
* Quality assurance
