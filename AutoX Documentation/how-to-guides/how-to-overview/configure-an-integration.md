# Configure an Integration

**Who can do this**

* Platform Admin
* Integration Engineer

&#x20;

**What is an Operation?**

An Operation defines a specific action within an integration, such as:

* GET customer details
* POST invoice data
* Trigger webhook

&#x20;

**Steps**

1. Open an existing Integration
2. Click Add Operation
3. Configure:
4. API Endpoint URL
5. HTTP Method
6. Request payload (if required)
7. Headers (key-value)
8. Review Request Preview
9. Click Save

&#x20;

**What Happens Next**

* Operation becomes selectable in Connector Tasks
* Request structure is validated at runtime

&#x20;

**Important Notes**

* Operations abstract technical API details
* Sensitive data is masked in logs
* Incorrect configuration leads to runtime failure
